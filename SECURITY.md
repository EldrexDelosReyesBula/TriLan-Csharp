# Security Policy

## Supported Versions

This section outlines which versions of TriLan C# are actively supported with **security updates**:

| Version | Supported          |
| ------- | ------------------ |
| 5.1.x   | :white_check_mark: |
| 5.0.x   | :x:                |
| 4.0.x   | :white_check_mark: |
| < 4.0   | :x:                |

> Versions marked with :white_check_mark: receive active security updates.  
> Versions marked with :x: are no longer supported and may contain unresolved security vulnerabilities.

---

## Reporting a Vulnerability

If you discover a security vulnerability in TriLan C#, please follow these steps to report it responsibly:

1. **Email the Security Team**  
   Send a detailed report to: **landecs.org@gmail.com**  
   Include:
   - A clear description of the vulnerability  
   - Steps to reproduce the issue  
   - Potential impact on users or data  
   - Screenshots or proof-of-concept if applicable  

2. **Acknowledgment**  
   You will receive a confirmation email within **48 hours** acknowledging your report.

3. **Response Timeline**  
   - TriLan C# team will evaluate the report and may reach out for additional information  
   - Security patches or mitigations will be released as soon as possible  
   - Updates will be communicated via GitHub releases and the project website  

4. **Public Disclosure**  
   - Do not publicly disclose the vulnerability until it has been addressed  
   - Once fixed, contributors who reported the issue will be credited in the release notes  

Thank you for helping us keep TriLan C# safe and secure!
